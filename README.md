# Проектная работа: Интернет-магазин на React (SPA)

<h2 align="center"><a  href="https://nadezhdamartin.github.io/stellar-burgers/#/">Посмотреть на Github Pages</a></h2>

Стек: React, React Router Dom, Redux Toolkit, TypeScript, SCSS, Storybook, Webpack, Jest, Cypress

Структура проекта:

- src/ — исходные файлы проекта
- src/components — папка с tsx компонентами
- src/components/ui — папка с ui-компонентами
- src/pages — папка с готовыми страницами
- src/services — папка с глобальным store и slices
- src/stories — папка с компонентами для Storybook
- src/utils — папка с api-функциями
- cypress — папка с Cypress-тестами
- tests — папка с Jest-тестами

Важные файлы:

- public/index.html — HTML-файл главной страницы
- src/index.tsx — точка входа приложения
- src/components/app.tsx — корневой компонент
- src/services/store.ts — файл с глобальным store-приложения
- src/services/slices — централизованное хранилище состояния приложения (Redux Toolkit):

| Путь к файлу | Назначение | Тип логики |
| :--- | :--- | :--- |
| `src/services/slices/constructorSlice` | Логика конструктора бургера | Локальный редюсер |
| `src/services/slices/feedSlice` | Данные ленты заказов | Асинхронный запрос (API) |
| `src/services/slices/ingredientsSlice` | Данные ингредиентов | Асинхронный запрос (API) |
| `src/services/slices/orderSlice` | Оформление заказа | Асинхронный запрос (API) |
| `src/services/slices/userOrdersSlice` | История заказов пользователя | Асинхронный запрос (API) |
| `src/services/slices/userSlice` | Авторизация и профиль | API + Редюсер |

## Установка и запуск

Для установки и запуска проекта необходимо выполнить команды


или

## Функциональные возможности

- На главной странице отображается конструктор и ингредиенты, незалогиненный пользователь может собирать бургер и видит ленту заказов.
- При нажатии на кнопку «Оформить заказ» или при попытке перейти в личный кабинет происходит проверка авторизации. Если пользователь не залогинен — перенаправление его на форму авторизации.
- На странице профиля пользователя можно редактировать данные в форме. При нажатии кнопки «Сохранить» отправляется соответствующий запрос к API, который содержит изменённые данные. При нажатии кнопки «Отмена» значения полей формы возвращаются в состояние до редактирования, а кнопки «Отмена» и «Сохранить» скрываются.
- При клике по ингредиенту открывается модальное окно с описанием ингредиента. При прямом переходе на этот маршрут открывается страница с информацией об ингредиенте.
- Заказы могут делать только авторизованные пользователи, все остальные действия доступны и неавторизованным.
- «Лента заказов» обновляется в режиме реального времени, когда авторизованный пользователь создаёт заказ. Содержит подсчёт стоимости каждого заказа, исходя из стоимости его ингредиентов. Отображает количество всех заказов, а также количество заказов за день.
- «История заказов» доступен только авторизованным пользователям. Обновляется в режиме реального времени и отображает статусы заказов. Содержит подсчёт стоимости каждого заказа, исходя из стоимости его ингредиентов.
- При клике по заказу открывается модальное окно с информацией о заказе. При прямом переходе открывается страница заказа.

| Путь | Описание |
|------|----------|
| `/` | Главная страница |
| `/feed` | Лента заказов |
| `/login` | Форма входа в приложение |
| `/register` | Форма регистрации |
| `/forgot-password` | Форма восстановления пароля |
| `/reset-password` | Форма ввода нового пароля |
| `/profile` | Профиль пользователя |
| `/profile/orders` | Заказы пользователя |
| `/feed/:number` | Заказ в модальном окне из ленты заказов |
| `/ingredients/:id` | Ингредиент в модальном окне из ленты заказов |
| `/profile/orders/:number` | Заказ пользователя в модальном окне из профиля пользователя |
| `*` | Страница не найдена|
